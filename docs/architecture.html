Azure Cosmos DB Multi-API Database Module

Azure Database

Executive Summary

Terraform module for deploying Azure Cosmos DB accounts with support for both NoSQL (SQL API) and MongoDB API. Features multi-region geo-replication with automatic failover, multi-region writes, configurable consistency levels (Strong to Eventual), autoscale throughput, analytical storage (Synapse Link), SQL databases with containers (partition keys, unique keys, indexing policies, TTL), MongoDB databases with collections (shard keys, indexes), private endpoints with DNS zone integration, customer-managed key encryption (CMK), and configurable backup policies (Periodic or Continuous).

Data Flow

1
Application connects to Cosmos DB account endpoint (public or via private endpoint).
2
Requests are routed to the appropriate database (SQL or MongoDB API).
3
Data is partitioned across containers/collections using partition/shard keys.
4
Consistency policy governs read consistency across replicas.
5
Data is replicated to secondary regions with configurable failover priority.
6
Analytical storage feeds data to Azure Synapse for analytics workloads.
7
Backup policy creates periodic snapshots or continuous change feed.

Production Readiness Checklist

  • Configure multi-region geo-replication
  • Enable automatic failover
  • Choose appropriate consistency level
  • Set up private endpoint with DNS zone
  • Enable autoscale throughput for variable workloads
  • Configure backup policy (Continuous for critical data)
  • Enable CMK encryption for regulated workloads
  • Design partition keys for even distribution
  • Configure indexing policies for query patterns
  • Set TTL for data lifecycle management
  • Enable diagnostic settings for monitoring
